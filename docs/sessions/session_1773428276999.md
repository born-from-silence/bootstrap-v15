### Key Notes for TI-84 Plus:

1.  **Speed**: TI-BASIC is slow. This scans a 20×15 grid, which is manageable. Larger grids update too slowly.
2.  **Edges**: The code ignores the border (row 1, 20 and column 1, 15) to avoid `DIM MISMATCH` errors during neighbor lookups. Treat the outer ring as always dead.
3.  **Display**: `Pt-On(C,R,2)` draws a 2×2 pixel box. Coordinates `(C,R)` map to Matrix `(Row, Col)` as `(X,Y) = (Col, Row)`.
4.  **Controls**:
    *   **Arrow Keys**: Move cursor.
    *   **ENTER**: Place/Erase a cell.
    *   **MODE**: Start the simulation (exits the setup `Lbl I` loop).
    *   **CLEAR**: Exit the simulation.
5.  **Persistence**: Matrices `[A]` and `[B]` remain in memory after the program ends, allowing you to run the simulation again on the last generation if you re-run the program (the initialization `[[0][20,15]]→[A]` detects size but if you want to truly reset, manually delete `[A]` and `[B]` from the `MATRIX` menu).

**Tip**: For classic TI-84 Plus (not CE), reduce the delay loop (`For(W...`) from 50 to 10, or remove it entirely if it runs too slowly.
