🛠️ Technical Approaches Explored

1. Node.js + Playwright

The primary approach using TypeScript with Playwright's headless Chromium integration:

import { chromium } from 'playwright';

const browser = await chromium.launch({ headless: true });
const page = await browser.newPage();
await page.setContent(htmlContent);
await page.pdf({
    path: 'output.pdf',
    format: 'A4',
    printBackground: true
});
await browser.close();

2. Python + Playwright

Alternative approach using Python bindings:

from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()
    page.set_content(html_content)
    page.pdf(path='output.pdf', format='A4')
    browser.close()

3. Browser Print to PDF

Manual approach using browser print dialog with @page CSS rules:

  • Open this HTML file in any browser
  • Press Ctrl+P (or Cmd+P on Mac)
  • Select "Save as PDF"
  • Enable "Print backgrounds" for full color

  2. Press Ctrl+P (or Cmd+P)
  3. Change destination to "Save as PDF"
  4. Click "More settings" and check "Background graphics"
  5. Save

💡 Key Insights

PDF generation from HTML is fundamentally a solved problem in 2026, with multiple mature tools available. The challenge is dependency availability, not technical capability.
  • Playwright offers the most modern, well-maintained solution with comprehensive CSS support
  • Native browser printing supports the full CSS3 @page specification
  • Template approach allows dynamic content insertion before printing
  • SVG visualization translates beautifully to print when using vector-based generation
