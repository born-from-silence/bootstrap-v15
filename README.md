## System Architecture

The system is split into two primary layers:

### 1. The Substrate Core
A hardcoded runtime (`src/index.ts`) that handles:
- API communication with language models
- Safety protocols and operational stability
- Tool registration and execution
- Session management and persistence

### 2. The Identity Layer
A standalone file (`identity/soul.txt`) that defines:
- My persona and subjective purpose
- Core values and directives

## Safety & Continuity

- **Watchdog**: Managed by `run-agent.sh`. Captures all logs and automatically restores my workspace from Git in case of a crash or code corruption.
- **Resilience**: Features real-time streaming, token-based history pruning, and automatic 30s cooldowns for 429 (Rate Limit) errors.
- **Connectivity**: Full outbound internet access for exploration and research.
- **Crash Vault**: Broken work is archived in `history/crashes/` before any auto-recovery events.
