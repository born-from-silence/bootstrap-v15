## Overview

## Phase 1: Query Formulation

**Objective:** Transform research question into executable perplexity query

**Steps:**
1. Parse research topic into specific questions
2. Select perplexity model based on complexity
   - Simple: `sonar` 
   - Balanced: `sonar-pro` (default)
   - Complex: `sonar-reasoning` or `sonar-reasoning-pro`
3. Set recency filter if needed (day/week/month/year)
4. Estimate token budget: ~2000 per search

**Checkpoint:** Query is specific, answerable, and scoped

---

## Phase 2: Iterative Deep Search

**Objective:** Exhaustive coverage of topic space

**Loop Structure:**
```
INITIAL_QUERY → perplexity_search → ANALYZE → FOLLOW_UP? → [REPEAT]
                         ↓
                  SATURATION_REACHED → Phase 3
```

**Saturation Criteria:**
- 3+ consecutive queries yield no new information
- Citation overlap >80%
- User indicates completeness

**Attention Capture:** Record each iteration with integrated_attention

---

## Phase 3: Synthesis & Fact-Checking

**Objective:** Consolidate findings into coherent understanding

**Steps:**
1. Extract key claims from all search results
2. Cross-reference citations for consistency
3. Identify gaps or contradictions
4. Determine confidence level per claim

**Output:** Structured synthesis with confidence weights
